# Presentation Outline

Target length: 5-10 minutes.

## 1. Project Goal

I built a CUDA/C++ reaction-diffusion simulator. The project produces evolving
image frames from the Gray-Scott model and records metrics over time. I chose it
because it is visual, easy to demonstrate, and based on a data-parallel stencil
that fits GPU hardware well.

## 2. What the Program Does

Show the generated frames in `proof/gpu_demo` or `proof/cpu_reference`.

Explain that each pixel stores two chemical concentrations, A and B. At each
time step, every pixel reads its neighbors, applies the Gray-Scott equations,
and writes the next value.

## 3. CUDA Technique

Open `src/gpu_backend.cu`.

Talk through:

- `UpdateKernel`
- 2D CUDA thread blocks
- one thread per grid cell
- nine-point Laplacian stencil
- double-buffered device memory
- CUDA event timing

## 4. Command-Line Demo

Run:

```bash
./run.sh
```

Then show:

- terminal output with backend, device name, compute capability, grid, steps,
  block size, and status
- `proof/gpu_demo/metrics.csv`
- several `frame_*.ppm` images

## 5. Results

Discuss how the pattern changes over time. The metrics file shows that the mean
concentration of chemical B changes as reaction structures expand across the
grid. The image frames provide visual proof that multiple time slices were
generated by one execution.

## 6. Challenges

Mention the read/write hazard in stencil code and how the program avoids it
with ping-pong buffers. Also mention that proof artifacts and CLI options were
important for making the project reviewable, not just runnable.

## 7. Next Steps

Good next steps would be shared-memory tiling, CPU-vs-GPU benchmark tables,
PNG/video export, and support for external seed images.
